Most accessible way to interact with Windows Internals is to interface through [[Windows API Calls]] . They provide native functionality to interact with the Windows OS.
Windows processor has a user and kernel mode , it switches between these 2 depending on the access and requested mode. 
The switch is facilitated by system and API calls.
